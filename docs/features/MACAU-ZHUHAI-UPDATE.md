# 澳门演出特殊处理更新

## 更新内容

当演出场馆位于澳门时，系统会自动提供珠海的酒店和机票信息，方便用户规划行程。

## 实现逻辑

### 1. 城市判断
- 在获取场馆信息后，检查场馆所在城市是否为澳门
- 判断条件：城市名包含"澳门"或等于"Macau"或"Macao"

### 2. 机票搜索
- **澳门演出**：搜索到珠海的机票
- **其他城市**：搜索到演出城市的机票

### 3. 酒店搜索
- **澳门演出**：搜索珠海市的酒店
- **其他城市**：搜索演出场馆周边的酒店

### 4. 高铁/巴士搜索
- **澳门演出**：搜索到珠海的高铁/巴士
- **其他城市**：搜索到演出城市的高铁/巴士

## 修改的文件

### cloudflare-pages/index.html
1. **State 定义** (行 1823)
   - 新增 `hotelSearchLocation` 字段，用于保存酒店搜索位置信息

2. **clearItinerary 函数** (行 1868)
   - 清除行程时重置 `hotelSearchLocation`

3. **generateItinerary 函数** (行 2462-2470)
   - 添加澳门判断逻辑
   - 当场馆在澳门时，将搜索城市设为珠海
   - 更新加载提示信息

4. **酒店搜索逻辑** (行 2529-2558)
   - 澳门演出时，通过 POI 搜索获取珠海市中心位置
   - 使用珠海位置搜索酒店
   - 将搜索位置信息保存到 `state.hotelSearchLocation`

5. **loadMoreHotels 函数** (行 3705-3720)
   - 使用保存的 `hotelSearchLocation` 进行扩展搜索
   - 确保"加载更多酒店"功能使用正确的搜索位置

## 用户体验

### 提示信息
当用户选择澳门的演出场馆时，系统会显示：
```
正在查询交通信息...
搜索机场和火车站（澳门演出，为您搜索珠海交通）
```

### 搜索逻辑
1. 用户输入澳门的演出场馆（如"澳门威尼斯人金光综艺馆"）
2. 系统识别场馆城市为"澳门"
3. 自动切换搜索目标：
   - 机票：搜索飞往珠海的航班
   - 高铁：搜索到珠海的高铁
   - 巴士：搜索到珠海的巴士
   - 酒店：搜索珠海市的酒店

## 降级处理

如果获取珠海位置信息失败，系统会：
- 在控制台记录警告信息
- 回退到使用澳门场馆位置进行搜索
- 确保功能不会完全失败

## 测试建议

### 测试场景
1. **澳门演出场馆**
   - 威尼斯人金光综艺馆
   - 澳门综艺馆
   - 澳门百老汇舞台

2. **验证点**
   - 机票目的地是否为珠海（ZUH）
   - 酒店是否显示珠海的酒店
   - 提示信息是否显示"澳门演出，为您搜索珠海交通"

3. **非澳门场馆**
   - 验证其他城市的演出不受影响
   - 确保正常搜索演出城市的交通和酒店
