## 技术验证

### 浏览器开发者工具检查

1. 打开浏览器开发者工具（F12）
2. 切换到 Console 标签页
3. 生成澳门演出行程
4. 检查是否有以下日志：
   - 如果成功获取珠海位置：无错误日志
   - 如果获取珠海位置失败：`获取珠海位置信息失败，使用澳门场馆位置`

### Network 面板检查

1. 切换到 Network 标签页
2. 生成澳门演出行程
3. 查找以下 API 请求：
   - `/api/hotel-search` - 请求体中的 `destination` 应为"珠海"
   - `/api/flight-search` - 请求体中的 `to_code` 应为珠海机场代码（ZUH）
   - `/api/city-airport` - 应查询"珠海"

## 边界情况测试

### 1. 场馆名称包含"澳门"
- **测试**：输入"澳门xxx场馆"
- **预期**：触发珠海搜索

### 2. 场馆名称为英文
- **测试**：输入场馆后，系统获取的城市信息为"Macau"或"Macao"
- **预期**：触发珠海搜索

### 3. POI 搜索失败场景
- **测试**：网络不稳定时生成行程
- **预期**：降级使用澳门场馆位置，功能不会完全失败

## 问题排查

如果功能异常，请检查：

1. **浏览器缓存**
   - 清除浏览器缓存后重试
   - 或使用隐私模式/无痕模式

2. **API 状态**
   - 检查 Network 面板中的 API 响应
   - 确认 `/api/poi-search` 返回正确的城市信息

3. **日志信息**
   - 查看 Console 中是否有错误日志
   - 特别关注与"珠海"相关的搜索日志

## 回滚方案

如果发现严重问题，可以通过以下方式回滚：

```bash
# 恢复到之前的版本
npx wrangler rollback --message "回滚澳门特殊处理"
```

或者从 Git 历史恢复文件：
```bash
git checkout HEAD~1 cloudflare-pages/index.html
cp cloudflare-pages/index.html public/index.html
npx wrangler deploy
```
