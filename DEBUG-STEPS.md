在终端运行：
```bash
wrangler tail
```

保持这个窗口打开！

### 步骤 2：重新测试订单创建

1. 打开浏览器
2. 访问：https://tripay-music-app.pages.dev/flights.html
3. 搜索航班
4. 选择航班
5. 填写完整信息：
   - ✅ 乘客姓名
   - ✅ 证件类型

   - ✅ 联系人姓名


## 🎯 我们要找的信息

日志会显示：
1. **收到的原始数据** - 查看前端发送了什么
2. **乘客信息** - 查看格式化后的数据
3. **缺少的字段** - 如果验证失败，会显示具体缺少哪些字段

---

## 📋 可能的情况

### 情况 A：字段名不匹配
如果看到：
```json
"passengers": [
  {
    "name": "张三",
    "idType": "id_card",     // ❌ 驼峰命名
    "idNumber": "xxx",       // ❌ 驼峰命名
    "phone": "xxx"
  }
]
```
→ 说明前端使用了驼峰命名

### 情况 B：字段为空字符串
```json
"passengers": [
  {
    "name": "张三",
    "id_type": "id_card",
    "id_number": "",         // ❌ 空字符串
    "phone": "xxx"
  }
]
```
→ 说明表单验证有问题

### 情况 C：字段被嵌套
```json
"passengers": [
  {
    "passenger": {           // ❌ 多了一层
      "name": "张三",
      ...
    }
  }
]
```
